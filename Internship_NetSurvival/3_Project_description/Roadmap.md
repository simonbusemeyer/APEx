# **Onboarding / Understanding (Weeks 1–2)**

- understand the **relative survival framework**: read key methodological papers (provided + others found through independent literature search),
- conduct a **clinical literature review**: high‑prognosis cancers in older adults, other possible ideas (e.g., thyroid cancer in young patients?),
- understand the **code structure** by modifying the provided code according to the instructions in *“code_modif_what_to_do”*.

**Deliverable (end of Week 2):**    
 A 2–3 page **literature review summary** + **operational modified code** with validated checks.

---

# **Simulation Design (Weeks 3–4)**

- propose **plausible scenarios**,
- discuss **parameter choices**,
- implement the **first simulation runs**,
- produce initial outputs (results for 2–3 pilot scenarios) including performance metrics: 
  - bias at *t* years (difference between estimated and theoretical net survival),
  - RMSE,
  - ECR (empirical coverage rate),
  - width of the 95% CI for net survival.

**Deliverable (end of Week 3):**    
 A 1–2 page **scenario design proposal** with justification of parameter choices.
